# Evidence Policy

Builder Lab OS is built around evidence-based progress.

A technical project becomes stronger when every claim is supported by clear proof.

## Valid evidence

- GitHub commits
- screenshots
- terminal logs
- test reports
- video recordings
- hardware photos
- validation checklists
- demo reports
- documentation updates

## Sensitive material not to publish

- passwords
- API keys
- private documents
- personal identity documents
- private chats
- bank or financial documents
- private certificates
- real security logs from networks without review
- other people's personal data

## Rule

Every project claim must have a maturity status and, when possible, supporting evidence.
